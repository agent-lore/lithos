## Codebase Patterns
- EdgeStore.adjust_weight atomically clamps to [0.0, 1.0] — no need for manual clamping
- MCP tool functions are closures inside `_register_tools()` — they capture `self` for access to all LithosServer attributes
- `server.mcp.get_tool(name)` returns a tool object whose `.fn` can be called directly in tests
- `_process_task_feedback` is a private method on LithosServer; import reinforcement functions lazily inside it
- spaced_rep_strength default is 0.0 (not 0.5) — only salience defaults to 0.5
- StatsStore has dedicated atomic increment methods: increment_cited, increment_ignored, increment_misleading, update_salience, update_spaced_rep_strength
- Namespace resolution: always read from KnowledgeManager._meta_cache[node_id].namespace, never re-derive from path
- Edge uniqueness is (from_id, to_id, type, namespace) — use upsert for create-or-update
- Test fixtures: edge_store and stats_store are defined locally per test file, not in conftest.py; use pytest_asyncio.fixture
- KnowledgeManager.create() populates _meta_cache automatically; use namespace= kwarg to set explicit namespace in tests
- Use `uv run` prefix for all commands (pytest, ruff, pyright) — no system-level make available in this environment
- StatsStore.get_node_stats() returns dict[str, object] — use isinstance guard for pyright when extracting typed fields
- test_retrieve.py uses a separate `seeded_config` fixture with pre-written frontmatter files and stable UUIDs (_ID1, _ID2, _ID3)
- _CachedMeta in knowledge.py now has a `status` field — populate it in all four _meta_cache assignments
- _CachedMetaView in scouts.py now has a `status` field — use `_passes_status_filter` to exclude quarantined nodes
- KnowledgeManager.update(id=..., lcma_status='quarantined', agent='lithos-enrich') sets status and updates _meta_cache
- list_all() accepts optional `exclude_status: list[str] | None` for filtering
- Pre-existing test failure: test_different_note_types_produce_different_scores — not related to reinforcement changes
- receipts table: id is TEXT PK, final_nodes is JSON string of [{id, ...}]; use ORDER BY rowid DESC for recency
- StatsStore.get_receipt and get_latest_receipt both return dict with `final_node_ids: list[str]` extracted from JSON

## 2026-04-12 - US-006
- What was implemented
  - Extended `lithos_task_complete` with three optional parameters: `cited_nodes`, `misleading_nodes`, `receipt_id`
  - Added `_process_task_feedback` private method to LithosServer for receipt validation and reinforcement
  - Receipt validation: exact receipt_id lookup with task_id check (error if mismatch); fallback to latest receipt (warning + drop if none)
  - Feedback IDs are intersected with receipt node IDs, then with existing knowledge (prevents writes for deleted notes)
  - Calls `reinforce_cited_nodes` + `reinforce_edges_between` for cited, `penalize_misleading` + `weaken_edges_for_bad_context` for misleading
  - Nodes in receipt but not cited/misleading are passed to `penalize_ignored`
  - `None` means no feedback (learning skipped); `[]` means explicit empty feedback (all receipt nodes ignored)
  - TASK_COMPLETED event payload includes `cited_nodes`, `misleading_nodes`, `receipt_id` as JSON-encoded strings via json.dumps
- Files changed
  - `src/lithos/server.py` (modified — added feedback params to lithos_task_complete + _process_task_feedback method)
  - `tests/test_task_complete_feedback.py` (new, 13 tests)
- **Learnings for future iterations:**
  - MCP tool functions are closures in `_register_tools()` — they capture `self`
  - Test pattern: use `server.mcp.get_tool(name).fn(**kwargs)` to call tools in tests
  - Import reinforcement functions lazily inside `_process_task_feedback` to avoid circular imports
  - `spaced_rep_strength` defaults to 0.0 (not 0.5), only `salience` defaults to 0.5
  - `knowledge._meta_cache` is the fastest way to check if a node exists (no I/O)

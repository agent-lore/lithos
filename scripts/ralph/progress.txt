## Codebase Patterns
- StatsStore uses `INSERT ... ON CONFLICT(node_id) DO UPDATE SET` for all upsert operations
- StatsStore methods follow pattern: `await self._ensure_open()`, then `async with aiosqlite.connect(self.db_path) as db`
- Schema migrations for adding columns use `PRAGMA table_info` check + `ALTER TABLE ADD COLUMN` in a static method called from `open()`
- Tests use `stats_store` fixture from `test_config`, assertions use `get_node_stats()` to verify state
- `uv run` replaces `make` when make is unavailable (e.g., `uv run ruff check`, `uv run pyright`, `uv run pytest`)

## Codebase Patterns
- EdgeStore methods follow pattern: `await self._ensure_open()`, then `async with aiosqlite.connect(self.db_path) as db:` with `db.row_factory = aiosqlite.Row` for named access
- EdgeStore returns dicts with explicit key mapping from Row objects (not `dict(r)` for type safety)
- Tests use `edge_store` fixture from `test_config: LithosConfig` — no manual setup needed
- `make` not available in container; use `uv run ruff check/format`, `uv run pyright`, `uv run pytest` directly
- Git identity needs to be set in container: `git config user.email/name`
- MCP tools registered via `@self.mcp.tool()` inside `_register_tools()` with access to `self.edge_store`, `self.knowledge`, etc.
- Tool error envelope: `{"status": "error", "code": "...", "message": "..."}`
- Tool event emission: `await self._emit(LithosEvent(type=EDGE_UPSERTED, payload={...}))`

## Codebase Patterns
- KnowledgeMetadata uses `_KNOWN_METADATA_KEYS` frozenset to separate known vs unknown (extra) fields — new metadata fields must be added there
- `from_dict` only unpacks what it receives; read-time defaults are applied by callers with path context (not in from_dict)
- `_resolve_safe_path` returns (relative_path, full_path) — relative_path is relative to knowledge_path
- `to_dict` conditionally includes optional fields (only when not None) to keep frontmatter clean
- Tests use `test_config` fixture (conftest.py) which creates isolated temp dirs; `knowledge_manager` fixture wraps it
- Use `fm.Post(content, **kwargs)` + `fm.dumps(post)` to write test notes manually to disk
- MCP boundary uses _UNSET sentinel: None from caller → _UNSET (preserve existing), value → pass through — used for all optional update fields
- Integration tests use `server` fixture and `_call_tool(server, name, args)` helper to invoke MCP tools via `server.mcp._call_tool_mcp()`
- LCMA enum constants: VALID_ACCESS_SCOPES, VALID_NOTE_TYPES, VALID_STATUSES exported from knowledge.py
- EdgeStore follows coordination.db pattern: `async def open()` creates schema, `aiosqlite.connect(db_path)` per operation, no persistent connection
- Corrupt DB detection uses `PRAGMA integrity_check` — `SELECT 1` doesn't read the file so can't detect corruption
- `StorageConfig.edges_db_path` property gives the canonical path: `data_dir / ".lithos" / "edges.db"`
- Quarantine naming: `edges.db.corrupt-{YYYYMMDDTHHMMSSZ}` with suffix disambiguation
- `StorageConfig.stats_db_path` property gives `data_dir / ".lithos" / "stats.db"`
- StatsStore follows the same pattern as EdgeStore: open(), _probe(), _quarantine(), aiosqlite per-op
- receipts table: `"limit"` column name is a SQL keyword — must be quoted in INSERT/SELECT statements
- receipts.namespace_filter is TEXT NULL (SQL NULL when None, JSON array string when provided)

- Event payloads use `dict[str, str | int | float | bool | None]` type â€” see `LithosEvent.payload`
- File-watcher delete handler emits NOTE_DELETED before calling knowledge.delete() to preserve doc_id access
- enrich_queue schema uses `attempts` column for retry tracking; drain methods filter by `attempts < max_enrich_attempts`

- Pre-existing format issue in tests/test_conflict_resolve.py â€” not introduced by this work
- StatsStore schema tests use `PRAGMA table_info(<table>)` to check columns as a set
- EnrichWorker subscribes with ENRICH_SUBSCRIBER_QUEUE_SIZE (10,000) to absorb burst events
- _resolve_node_id() is the central normalization contract for mapping event payloads to node IDs
- note.deleted events do NOT check KnowledgeManager existence (node already deleted)
- StatsStore.drain_pending_nodes/tasks accept max_attempts kwarg; requeue_failed increments attempts
- Server fixture in conftest.py must call `await srv.stop_enrich_worker()` before `stop_file_watcher()`
- EnrichWorker tests use `asyncio.sleep(0.1)` to let consumer process events from the queue
- StatsStore migrations follow pattern: static method checks PRAGMA table_info, ALTER TABLE if column missing
- _project_node_provenance handles both existing (sync edges) and deleted (remove all) nodes
- Decay is convergent via last_decay_applied_at date check â€” safe to run multiple times per day
- knowledge.get_doc_sources(doc_id) returns list[str] of source IDs (public API for _doc_to_sources)
- knowledge._meta_cache[doc_id].namespace gives resolved namespace for edge operations
- Cross-DB atomicity (stats.db â†’ edges.db): use record-before-write pattern for at-most-once semantics
- Same-DB atomicity (multiple stats.db writes): combine in single aiosqlite transaction
- Pyright walrus operator trick: `isinstance((v := d.get("k")), int) and v >= 2` narrows correctly
- Adding a metadata field: _KNOWN_METADATA_KEYS, KnowledgeMetadata dataclass, to_dict, from_dict, update() param
- Entity extraction: _extract_entities_from_text is a pure function; _extract_entities on EnrichWorker reads/updates via KnowledgeManager
- mock_knowledge fixture needs `km.read = AsyncMock(...)` for tests where _enrich_node triggers entity extraction
- KnowledgeManager.create() returns WriteResult with .document.id (not .id directly)
